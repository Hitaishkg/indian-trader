## Purpose

Determines the current market regime by comparing the Nifty 50 close price
to its 200-day simple moving average. Adjusts position sizing for new trades
and signals stop-loss tightening for open positions based on how long the
index has been below its 200 DMA.

This is the final strategy filter before backtest integration. It sits
between `momentum.py` (ranked candidates) and the eventual
`backtest/runner.py` (Phase 2 step 5).

## Data Classes

### RegimeResult (frozen dataclass)

```python
@dataclass(frozen=True)
class RegimeResult:
    """Result of regime filter computation.

    Attributes:
        regime: One of "ABOVE_200DMA", "BELOW_200DMA", "BELOW_200DMA_10DAYS".
        nifty_close: Latest Nifty 50 closing price (INR).
        sma_200: Current 200-day SMA value of Nifty 50 close.
        consecutive_days_below: Number of consecutive trading days (from most
            recent) that Nifty 50 close has been below the rolling 200-day SMA.
        position_size_multiplier: 1.0 (above), 0.5 (below < 10 days),
            0.0 (below >= 10 days).
        tighten_stops: True when Nifty is below 200 DMA (any duration).
            Means open positions should use 1x ATR stop instead of 2x ATR.
        stop_tighten_symbols: List of symbol strings from open_positions
            that need their stop-loss tightened. Empty if tighten_stops
            is False or no open positions provided.
        computed_at_ist: ISO 8601 IST timestamp.
    """
    regime: str
    nifty_close: float
    sma_200: float
    consecutive_days_below: int
    position_size_multiplier: float
    tighten_stops: bool
    stop_tighten_symbols: list[str]
    computed_at_ist: str
```

## Public API

### 1. `apply_regime_filter`

```python
def apply_regime_filter(
    ranked_df: pd.DataFrame,
    nifty_ohlcv_df: pd.DataFrame,
    open_positions: list[dict[str, object]] | None = None,
) -> tuple[pd.DataFrame, RegimeResult]:
    """Apply Nifty 50 200 DMA regime filter to momentum-ranked candidates.

    Computes the 200-day SMA of Nifty 50 close prices, determines the
    current regime, adjusts position sizing for new trades, and signals
    stop-loss tightening for open positions.

    Args:
        ranked_df: Output of compute_momentum(). One row per momentum-
            ranked candidate. Required columns: symbol, momentum_score,
            twelve_month_return, one_month_return, rank, pct_from_52w_high,
            within_30pct_of_52w_high.
        nifty_ohlcv_df: Nifty 50 index OHLCV data. Required columns:
            date, close. Must have >= 200 rows to compute 200-day SMA.
            Rows should be sorted by date ascending (function will sort
            if not already sorted).
        open_positions: List of position dicts from PaperTrader.get_positions().
            Each dict must have at minimum a "symbol" key. If None or empty,
            stop_tighten_symbols will be [].

    Returns:
        Tuple of (filtered_df, regime_result).

        filtered_df: ranked_df with an added column "position_size_multiplier"
            (float). When regime is BELOW_200DMA_10DAYS, returns an empty
            DataFrame with the same columns (no candidates pass through).
            When regime is BELOW_200DMA, multiplier = 0.5. When ABOVE_200DMA,
            multiplier = 1.0. Original rank order is preserved.

        regime_result: RegimeResult frozen dataclass with full regime details.

    Raises:
        ValueError: If nifty_ohlcv_df has fewer than SMA_PERIOD (200) rows.
        ValueError: If nifty_ohlcv_df is missing required columns ("date", "close").
        ValueError: If ranked_df is missing required columns (symbol, momentum_score,
            twelve_month_return, one_month_return, rank, pct_from_52w_high,
            within_30pct_of_52w_high).
    """
```

**Behavior details:**

1. Sort `nifty_ohlcv_df` by date ascending if not already sorted.
2. Call `compute_200dma(nifty_ohlcv_df)` to get the SMA value.
3. Call `count_consecutive_days_below_200dma(nifty_ohlcv_df)` to get count.
4. Determine regime:
   - If latest close >= sma_200: `ABOVE_200DMA`
   - If latest close < sma_200 and consecutive_days < BELOW_DMA_BLOCK_DAYS: `BELOW_200DMA`
   - If latest close < sma_200 and consecutive_days >= BELOW_DMA_BLOCK_DAYS: `BELOW_200DMA_10DAYS`
5. Set position_size_multiplier from regime.
6. Set tighten_stops = True if regime is not ABOVE_200DMA.
7. Build stop_tighten_symbols: if tighten_stops is True and open_positions is not None/empty, extract all "symbol" values from open_positions.
8. Build filtered_df:
   - If ABOVE_200DMA or BELOW_200DMA: copy ranked_df, add "position_size_multiplier" column.
   - If BELOW_200DMA_10DAYS: return empty DataFrame with all expected columns including "position_size_multiplier".
9. If ranked_df is empty (0 rows): still compute regime and return empty DataFrame with correct columns + RegimeResult.
10. Log via `log_agent_action()`: regime status, nifty_close, sma_200, consecutive_days_below, number of candidates passed through, number of symbols needing stop tightening.

**Edge case -- Nifty exactly on 200 DMA:** When close == sma_200, regime is
`ABOVE_200DMA` (use `>=` comparison). This is a deliberate choice to avoid
unnecessary position reduction on a boundary value.

**Empty ranked_df:** Valid input. Returns empty DataFrame with correct schema
and a fully populated RegimeResult. No ValueError.

### 2. `compute_200dma`

```python
def compute_200dma(nifty_ohlcv_df: pd.DataFrame) -> float:
    """Compute the 200-day simple moving average of Nifty 50 close prices.

    Args:
        nifty_ohlcv_df: Nifty 50 OHLCV data with columns "date" and "close".
            Must have >= 200 rows.

    Returns:
        The 200-day SMA as a float (last value of the rolling mean).

    Raises:
        ValueError: If nifty_ohlcv_df has fewer than 200 rows.
        ValueError: If "close" column is missing.
    """
```

**Implementation:**
- Sort by date ascending.
- Compute `nifty_ohlcv_df["close"].rolling(window=SMA_PERIOD).mean()`.
- Return the last non-NaN value as float.
- Raise ValueError if fewer than SMA_PERIOD rows.

### 3. `count_consecutive_days_below_200dma`

```python
def count_consecutive_days_below_200dma(nifty_ohlcv_df: pd.DataFrame) -> int:
    """Count consecutive trading days the Nifty 50 close has been below the rolling 200-day SMA.

    Counts backwards from the most recent row. Stops counting when a day
    is found where close >= rolling 200-day SMA, or when the rolling SMA
    is NaN (first 199 rows).

    Args:
        nifty_ohlcv_df: Nifty 50 OHLCV data with columns "date" and "close".
            Must have >= 200 rows.

    Returns:
        Number of consecutive trading days (from most recent) that the close
        was strictly below the rolling 200-day SMA. Returns 0 if the latest
        close is >= the 200-day SMA.

    Raises:
        ValueError: If nifty_ohlcv_df has fewer than 200 rows.
        ValueError: If "close" column is missing.
    """
```

**Implementation:**
- Sort by date ascending.
- Compute rolling 200-day SMA for the full series.
- Starting from the last row, walk backwards counting rows where close < sma.
- Stop when close >= sma or sma is NaN. Return the count.
- If the latest row has close >= sma, return 0.

## Error Handling

| Condition | Error |
|-----------|-------|
| nifty_ohlcv_df has < 200 rows | `ValueError("nifty_ohlcv_df must have >= 200 rows, got N")` |
| nifty_ohlcv_df missing "close" column | `ValueError("nifty_ohlcv_df missing required columns: ['close']")` |
| nifty_ohlcv_df missing "date" column | `ValueError("nifty_ohlcv_df missing required columns: ['date']")` |
| ranked_df missing required columns | `ValueError("ranked_df missing required columns: [...]")` |
| Empty ranked_df (0 rows) | NOT an error. Return empty filtered_df with correct schema + valid RegimeResult |
| open_positions is None | NOT an error. stop_tighten_symbols = [] |
| open_positions is empty list | NOT an error. stop_tighten_symbols = [] |

No bare except clauses. Only ValueError is raised by this module.

## Integration Notes

### How regime.py fits in the pipeline

```
quality_filter.py -> momentum.py -> regime.py -> [backtest/runner.py]
                                       |
                                       +-> reads open_positions from PaperTrader
```

### Usage pattern (in main.py or orchestrator)

```python
from src.strategy.regime import apply_regime_filter, RegimeResult

# After momentum ranking
ranked_df, momentum_report = compute_momentum(quality_df, ohlcv_df)

# Fetch Nifty 50 OHLCV (need >= 200 rows)
nifty_ohlcv = fetch_ohlcv(["NIFTY 50"], start_date, end_date)

# Get open positions (if any)
paper_trader = PaperTrader()
positions = paper_trader.get_positions()

# Apply regime filter
filtered_df, regime_result = apply_regime_filter(ranked_df, nifty_ohlcv, positions)

# Use filtered_df for further processing
# Use regime_result.tighten_stops and regime_result.stop_tighten_symbols
# to call paper_trader.update_stop_loss() on affected positions
```

Note: regime.py does NOT call `PaperTrader.update_stop_loss()` itself. It only
identifies which symbols need tightening. The caller (main.py or orchestrator)
is responsible for executing the stop-loss updates. This keeps regime.py as a
pure computation module with no side effects beyond logging.
