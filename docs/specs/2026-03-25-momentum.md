## 1. Purpose

Computes the 12-1 momentum factor for quality-filtered stocks and selects the
top N candidates for the weekly trading watchlist. This is the second gate in
the three-step stock selection process:

  quality filter -> **momentum rank** -> regime filter

The 12-1 momentum factor is academically validated as the strongest single
return predictor in Indian equities (IIM Ahmedabad Fama-French-Momentum
research, NSE data since 1994). The "minus 1" removes short-term reversal
noise and captures the underlying directional trend.

This module is a pure calculation engine. It does not enforce recalculation
schedules (that is the orchestrator's responsibility), does not write to any
database table directly, and does not make network calls.

## 3. Public API

### 3.1 `compute_momentum(quality_df, ohlcv_df, top_n=5) -> tuple[pd.DataFrame, MomentumReport]`

```python
def compute_momentum(
    quality_df: pd.DataFrame,
    ohlcv_df: pd.DataFrame,
    top_n: int = 5,
) -> tuple[pd.DataFrame, MomentumReport]:
    """Compute 12-1 momentum scores and select top N candidates.

    Formula: momentum_score = 12-month total return - 1-month total return

    Where:
    - 12-month return = (close[today] - close[252 days ago]) / close[252 days ago]
    - 1-month return = (close[today] - close[21 days ago]) / close[21 days ago]

    Symbols with fewer than 252 trading days of history are excluded from
    ranking and logged as insufficient_history.

    Args:
        quality_df: Output of apply_quality_filter(). One row per quality-
            filtered symbol. Required columns: symbol, pct_from_52w_high,
            within_30pct_of_52w_high.
        ohlcv_df: Full OHLCV history. Multi-symbol DataFrame. Required
            columns: symbol, date, close. Date must be timezone-aware
            (Asia/Kolkata).
        top_n: Number of top candidates to return. Default 5.

    Returns:
        Tuple of (ranked_df, report).
        ranked_df: DataFrame with one row per selected candidate, sorted by
            rank ascending (rank 1 = highest momentum). Columns: symbol,
            momentum_score, twelve_month_return, one_month_return, rank,
            pct_from_52w_high, within_30pct_of_52w_high.
        report: MomentumReport frozen dataclass with computation metadata.

    Raises:
        ValueError: If quality_df is empty.
        ValueError: If ohlcv_df is empty.
        ValueError: If required columns missing from quality_df.
        ValueError: If required columns missing from ohlcv_df.
        ValueError: If top_n < 1.
    """
```

### 3.2 `class MomentumReport`

```python
@dataclass(frozen=True)
class MomentumReport:
    """Summary of momentum computation results.

    Attributes:
        scored_count: Number of symbols for which a valid momentum score
            was computed (had >= 252 rows of history).
        selected_count: Number of symbols in the final top-N output.
        insufficient_history_count: Number of symbols excluded for having
            < 252 rows of OHLCV history.
        tiebreaker_applied_count: Number of times the 2% tiebreaker rule
            was applied during ranking.
        computed_at_ist: ISO 8601 IST timestamp when computation was performed.
    """
    scored_count: int
    selected_count: int
    insufficient_history_count: int
    tiebreaker_applied_count: int
    computed_at_ist: str
```

## 5. Output DataFrame Schema

One row per selected candidate. If fewer than `top_n` symbols have valid
momentum scores, all scored symbols are returned (could be fewer than top_n).
If zero symbols have valid scores, return an empty DataFrame with the correct
column schema.

| Column | Type | Derivation |
|--------|------|-----------|
| symbol | str | From quality_df |
| momentum_score | float64 | twelve_month_return - one_month_return |
| twelve_month_return | float64 | (close_today - close_252d_ago) / close_252d_ago |
| one_month_return | float64 | (close_today - close_21d_ago) / close_21d_ago |
| rank | int64 | 1 = highest momentum score. Tiebreaker-adjusted. |
| pct_from_52w_high | float64 | Carried from quality_df |
| within_30pct_of_52w_high | bool | Carried from quality_df |

Sorted by `rank` ascending. RangeIndex.

## 6. Algorithm — Step by Step

### 6.1 Input validation

1. Raise `ValueError` if `quality_df` is empty.
2. Raise `ValueError` if `ohlcv_df` is empty.
3. Raise `ValueError` if `quality_df` is missing any of: `symbol`,
   `pct_from_52w_high`, `within_30pct_of_52w_high`.
4. Raise `ValueError` if `ohlcv_df` is missing any of: `symbol`, `date`, `close`.
5. Raise `ValueError` if `top_n < 1`.

### 6.2 Compute momentum score per symbol

For each symbol in `quality_df`:

1. Extract that symbol's rows from `ohlcv_df`.
2. Sort by `date` ascending (oldest first).
3. Check row count:
   - If `len(rows) < 252` -> assign `NaN` to momentum_score, twelve_month_return,
     one_month_return. Log as `insufficient_history` via `log_agent_action()`.
     Skip to next symbol.
4. Identify the most recent row as `today`. Identify the row at index
   `len(rows) - 252` as `twelve_months_ago`. Identify the row at index
   `len(rows) - 21` as `one_month_ago`.
   - **Clarification**: use positional indexing from the sorted array. The row
     at position `[-1]` is today, `[-252]` is 252 trading days ago, `[-21]` is
     21 trading days ago.
5. Extract close prices:
   - `close_today = rows.iloc[-1]["close"]`
   - `close_12m_ago = rows.iloc[-252]["close"]`
   - `close_1m_ago = rows.iloc[-21]["close"]`
6. Division-by-zero guard:
   - If `close_12m_ago == 0.0` -> assign NaN, log as `zero_close_12m`, skip.
   - If `close_1m_ago == 0.0` -> assign NaN, log as `zero_close_1m`, skip.
7. Compute returns:
   - `twelve_month_return = (close_today - close_12m_ago) / close_12m_ago`
   - `one_month_return = (close_today - close_1m_ago) / close_1m_ago`
8. Compute momentum score:
   - `momentum_score = twelve_month_return - one_month_return`
9. Log the computed score via `log_agent_action()`:
   agent_name="momentum", action=f"momentum_score={momentum_score:.4f}
   (12m={twelve_month_return:.4f}, 1m={one_month_return:.4f})",
   symbol=symbol, result="scored".

### 6.3 Filter and rank

1. Remove all symbols with NaN momentum_score from the ranking pool.
2. Sort remaining symbols by `momentum_score` descending.
3. Apply the 2% tiebreaker rule (see section 6.4).
4. Assign ranks starting from 1.
5. Take the top `top_n` symbols (or all if fewer than `top_n` scored).

### 6.4 Tiebreaker rule

After sorting by momentum_score descending, iterate through adjacent pairs in
the sorted list. For each pair (symbol_a at position i, symbol_b at position i+1):

1. Compute relative difference:
   `rel_diff = abs(score_a - score_b) / abs(score_b)` if `score_b != 0.0`.
   If `score_b == 0.0`, compute `rel_diff = abs(score_a)`. If both are zero,
   `rel_diff = 0.0`.
2. If `rel_diff < 0.02` (within 2%):
   - Compare `pct_from_52w_high` for both symbols.
   - The symbol with the **lower** `pct_from_52w_high` (closer to its 52-week
     high) wins the tiebreaker and takes the higher rank position.
   - If needed, swap positions in the sorted list.
   - Log the tiebreaker via `log_agent_action()`:
     agent_name="momentum",
     action=f"tiebreaker: {winner} (pct_from_52w_high={winner_pct:.4f}) beats
     {loser} (pct_from_52w_high={loser_pct:.4f}), scores within 2%",
     result="tiebreaker_applied".
   - Increment the tiebreaker counter.
3. If `rel_diff >= 0.02`: no swap, keep original order.

**Important**: the tiebreaker is applied as a single pass through adjacent pairs
after the initial sort. It is NOT a full re-sort. This matches the spec intent:
tiebreakers only matter when scores are nearly identical.

### 6.5 Build output

1. From the ranked list (after tiebreaker), take the first `top_n` symbols.
2. Merge with `quality_df` to carry over `pct_from_52w_high` and
   `within_30pct_of_52w_high`.
3. Build the output DataFrame with columns: symbol, momentum_score,
   twelve_month_return, one_month_return, rank, pct_from_52w_high,
   within_30pct_of_52w_high.
4. Sort by rank ascending, reset index.

### 6.6 Build MomentumReport

```python
MomentumReport(
    scored_count=<number of symbols with valid momentum scores>,
    selected_count=<number of symbols in output DataFrame>,
    insufficient_history_count=<number of symbols with < 252 rows>,
    tiebreaker_applied_count=<number of tiebreaker swaps made>,
    computed_at_ist=<IST ISO 8601 timestamp>,
)
```

### 6.7 Logging

Summary log after all computation:
- agent_name="momentum",
  action=f"momentum complete: {scored_count} scored, {selected_count} selected,
  {insufficient_history_count} insufficient history,
  {tiebreaker_applied_count} tiebreakers",
  result="ok".

Log the final selected symbols:
- agent_name="momentum",
  action=f"selected: rank={rank}, momentum_score={score:.4f}",
  symbol=symbol, result="selected".

## 7. Error Handling

| Scenario | Behaviour |
|----------|----------|
| `quality_df` is empty | Raise `ValueError("quality_df must not be empty")` |
| `ohlcv_df` is empty | Raise `ValueError("ohlcv_df must not be empty")` |
| Required column missing from quality_df | Raise `ValueError("quality_df missing required columns: {list}")` |
| Required column missing from ohlcv_df | Raise `ValueError("ohlcv_df missing required columns: {list}")` |
| `top_n < 1` | Raise `ValueError("top_n must be >= 1, got {top_n}")` |
| Symbol in quality_df has no rows in ohlcv_df | Treated as insufficient history (< 252 rows), NaN score, excluded |
| Symbol has < 252 rows of OHLCV | NaN score, excluded from ranking, logged as `insufficient_history` |
| `close == 0.0` at 252-day lookback | NaN score, excluded, logged as `zero_close_12m` |
| `close == 0.0` at 21-day lookback | NaN score, excluded, logged as `zero_close_1m` |
| All symbols have insufficient history | Return empty DataFrame + MomentumReport(scored_count=0, selected_count=0) |
| Fewer scored symbols than top_n | Return all scored symbols (selected_count < top_n) |
| `log_agent_action()` fails internally | Never raises (logger design) — no handling needed |

No bare `except` clauses. Only `ValueError` is raised by this module.

## 11. Test Hints (minimum 14)

### Happy path
1. **5 stocks all with sufficient history**: all 5 get valid scores, ranked 1-5, output has 5 rows.
2. **Top 3 only requested (top_n=3)**: 5 scored stocks, output has 3 rows with ranks 1-3.
3. **Momentum score computed correctly**: stock with close_today=120, close_252d_ago=100, close_21d_ago=115 -> twelve_month_return=0.20, one_month_return=0.0435, momentum_score=0.1565.
4. **Ranking order correct**: stock A score=0.30, stock B score=0.20 -> A gets rank 1, B gets rank 2.
5. **Output columns match spec**: output DataFrame has exactly 7 columns in the expected order.

### Tiebreaker
6. **Tiebreaker invoked (within 2%)**: stock A score=1.00, stock B score=0.99 (rel_diff=0.01 < 0.02). Stock B has pct_from_52w_high=0.05, stock A has 0.15. Stock B wins, gets higher rank.
7. **Tiebreaker NOT invoked (outside 2%)**: stock A score=1.00, stock B score=0.95 (rel_diff=0.05 >= 0.02). No swap — A stays ahead of B.
8. **Tiebreaker with equal pct_from_52w_high**: both within 2% and same pct_from_52w_high -> original order preserved (stable).
9. **Tiebreaker count in report**: 2 tiebreakers applied -> MomentumReport.tiebreaker_applied_count == 2.

### Insufficient history
10. **Symbol with < 252 rows excluded**: symbol has 200 rows -> excluded from output, insufficient_history_count=1 in report.
11. **Symbol with 0 rows in ohlcv_df**: symbol in quality_df but not in ohlcv_df -> excluded, counted as insufficient_history.
12. **All symbols insufficient history**: all have < 252 rows -> empty output, scored_count=0, selected_count=0.
13. **Mix of sufficient and insufficient**: 3 of 5 have enough history -> scored_count=3, output has min(top_n, 3) rows.

### Division by zero
14. **close_12m_ago is 0.0**: score is NaN, symbol excluded, logged as `zero_close_12m`.
15. **close_1m_ago is 0.0**: score is NaN, symbol excluded, logged as `zero_close_1m`.

### Input validation
16. **Empty quality_df**: raises `ValueError`.
17. **Empty ohlcv_df**: raises `ValueError`.
18. **Missing required column in quality_df**: raises `ValueError` naming the missing column(s).
19. **Missing required column in ohlcv_df**: raises `ValueError` naming the missing column(s).
20. **top_n = 0**: raises `ValueError`.
21. **top_n negative**: raises `ValueError`.

### MomentumReport
22. **MomentumReport is frozen**: attempting to mutate any attribute raises `FrozenInstanceError`.
23. **computed_at_ist is valid IST timestamp**: parseable as ISO 8601 with +05:30 offset.
24. **Report counts sum correctly**: scored_count + insufficient_history_count == total symbols in quality_df.

### Edge cases
25. **Exactly 252 rows**: symbol with exactly 252 rows gets a valid score (boundary).
26. **Fewer scored than top_n**: only 2 symbols scored, top_n=5 -> output has 2 rows, selected_count=2.
27. **Negative momentum scores**: stock declined over 12 months -> momentum_score is negative, still ranked correctly (less negative = higher rank).
28. **Tiebreaker with score_b == 0**: score_a=0.01, score_b=0.0 -> rel_diff = abs(0.01) = 0.01 < 0.02 -> tiebreaker applied.

## 12. Non-Goals

- This module does NOT enforce weekly recalculation schedule (that is the
  orchestrator's responsibility). It computes on whatever data it receives.
- This module does NOT apply the regime filter (that is `regime.py`).
- This module does NOT apply quality filters (that is `quality_filter.py`).
- This module does NOT fetch data — it receives DataFrames as arguments.
- This module does NOT write to any database table other than `agent_logs` via logger.
- This module does NOT send notifications.
- This module does NOT compute technical indicators (RSI, MACD, etc.).
- The `within_30pct_of_52w_high` column is carried through to output but
  is only used for tiebreaking. It is NOT used for elimination.
