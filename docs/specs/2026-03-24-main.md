`setup_logging()` must be called before any module that produces log output.

**Rationale**: `settings` is imported by `logger.py` internally, so it will

## 9. ATR Calculation (inline, no external dependency)

Compute 14-period ATR using the cleaned OHLCV data for the stock selected
for the paper trade. Do NOT import `src/indicators/technical.py` — it does
not exist yet (Phase 2).

### Implementation

```python
import pandas as pd

def compute_atr(symbol_df: pd.DataFrame, period: int = 14) -> float | None:
    """Compute Average True Range for a single symbol's OHLCV data.

    Uses the standard Wilder smoothing method: TR = max(H-L, |H-Cp|, |L-Cp|),
    then exponential moving average over `period` bars.

    Args:
        symbol_df: OHLCV DataFrame for ONE symbol, sorted by date ascending.
                   Must have columns: high, low, close.
        period: ATR lookback period. Default 14.

    Returns:
        ATR value as float, or None if insufficient data (< period+1 rows).
    """
    if len(symbol_df) < period + 1:
        return None
    high = symbol_df["high"]
    low = symbol_df["low"]
    close_prev = symbol_df["close"].shift(1)
    tr = pd.concat([
        high - low,
        (high - close_prev).abs(),
        (low - close_prev).abs(),
    ], axis=1).max(axis=1)
    atr = tr.ewm(span=period, adjust=False).mean().iloc[-1]
    return float(atr)
```

This function is defined inside `main.py`. It will be superseded by
`src/indicators/technical.py` in Phase 2. The Coder Agent should define it
as a module-level function (not nested inside `main()`), with the docstring
and type hints shown above.

**Design decision**: Using `ewm(span=period)` for Wilder smoothing. This is
mathematically equivalent to `ewm(alpha=1/period)` and is the standard
pandas approach. The exact ATR value is not critical here — it is only used
to set reasonable stop-loss/take-profit levels on a simulated paper trade.

## 10. Paper Trade Placement

### Stock selection

Use the **first symbol** from `SYMBOLS` that has data in the validated OHLCV
DataFrame. In practice this will be `RELIANCE` (first in the list). If
`RELIANCE` has no data after validation, iterate through the list until one
is found.

### Price and level calculation

```python
# Filter cleaned_df for the selected symbol
symbol_df = cleaned_df[cleaned_df["symbol"] == selected_symbol].sort_values("date")

# Entry price = last close
entry_price = float(symbol_df["close"].iloc[-1])

# ATR calculation
atr = compute_atr(symbol_df)

if atr is not None and atr > 0:
    # ATR-based stop-loss and take-profit
    raw_stop_distance = atr * 2.0
    # Hard cap: stop distance cannot exceed 3% of entry price
    max_stop_distance = entry_price * 0.03
    stop_distance = min(raw_stop_distance, max_stop_distance)
    stop_loss = round(entry_price - stop_distance, 2)
    # Take-profit: minimum 1:2 risk-reward
    take_profit = round(entry_price + (stop_distance * 2.0), 2)
else:
    # Fallback: 2% stop-loss, 4% take-profit
    stop_loss = round(entry_price * 0.98, 2)
    take_profit = round(entry_price * 1.04, 2)

# Fixed quantity for dry-run
quantity = 1
```

### Placement

```python
trader = PaperTrader()
order_id = trader.place_order(
    symbol=selected_symbol,
    side="BUY",
    quantity=quantity,
    entry_price=entry_price,
    stop_loss=stop_loss,
    take_profit=take_profit,
)
```

### P&L query

```python
pnl = trader.get_pnl()
positions = trader.get_positions()
```

**Design decisions**:
- **Quantity = 1**: Fixed at 1 share. Position sizing logic lives in Phase 2.
  For a dry-run, 1 share keeps the order value well under MAX_TRADE_AMOUNT
  for all 5 stocks.
- **Stop-loss 3% hard cap**: Matches the strategy rule in `strategy.md`.
- **ATR fallback**: If ATR cannot be computed (insufficient data), use
  simple 2%/4% levels. This should not happen with 30 days of data and
  14-period ATR, but the fallback handles edge cases.
- **Round to 2 decimal places**: INR prices have paisa precision.

## 11. Error Handling

The script uses a top-level try/except structure with specific exception types.

### Structure

```python
def main() -> None:
    try:
        # ... pipeline steps ...
    except ConfigurationError as exc:
        # settings failed to load — already printed by settings.py
        log_agent_action("main", f"Configuration error: {exc}", level="ERROR", result="error")
        send_alert("Pipeline failed", f"Configuration error: {exc}")
        sys.exit(1)
    except FetchError as exc:
        log_agent_action("main", f"Data fetch failed: {exc}", level="ERROR", result="error")
        send_alert("Pipeline failed", f"Data fetch failed: {exc}")
        sys.exit(1)
    except DataQualityError as exc:
        log_agent_action(
            "main",
            f"Data quality below threshold: score={exc.universe_quality_score:.4f}",
            level="ERROR",
            result="error",
            data_quality_score=exc.universe_quality_score,
        )
        send_alert("Pipeline failed", f"Data quality score {exc.universe_quality_score:.4f} below 0.60 threshold")
        sys.exit(1)
    except ValueError as exc:
        log_agent_action("main", f"Validation error: {exc}", level="ERROR", result="error")
        send_alert("Pipeline failed", f"Validation error: {exc}")
        sys.exit(1)
```

**Important**: The `ConfigurationError` import is needed. Import it from
`src.config.settings`. However, because `settings` is a module-level singleton
that loads at import time, a `ConfigurationError` will actually fire during
the import phase — before `main()` is entered. The try/except around it is
still useful if `main.py` is imported as a module rather than run directly.
To handle the import-time failure cleanly, wrap the settings import in its
own try/except at the top of the file:

```python
try:
    from src.config.settings import settings, ConfigurationError
except Exception as exc:
    print(f"[main] Fatal: configuration failed: {exc}", file=sys.stderr)
    sys.exit(1)
```

This guarantees `main.py` never produces an unhandled traceback for missing
.env variables.

### No bare except clauses

Every except clause names a specific exception type. The only exception is
the settings import guard at the top level, which uses `Exception` (not bare
`except:`) to catch `ConfigurationError` and any unexpected import errors.

## 15. Idempotency

Each run places a new paper trade. This is acceptable for a dry-run milestone
script. The `PaperTrader.place_order()` method will raise `ValueError` if a
position for the same symbol is already open (UNIQUE constraint on
`positions.symbol`). To handle re-runs gracefully:

Before placing the order, check if a position already exists for the selected
symbol. If it does, skip the trade placement, log it as
`"Position already open for {symbol}, skipping paper trade"`, and proceed
directly to the P&L query and summary.

```python
existing_positions = trader.get_positions()
already_open = any(p["symbol"] == selected_symbol for p in existing_positions)

if already_open:
    log_agent_action("main", f"Position already open for {selected_symbol}, skipping trade", level="INFO")
    order_id = "N/A (position already open)"
else:
    order_id = trader.place_order(...)
```

## 21. Design Decisions Made by Architect

These were not explicitly stated in the requirements and need approval:

1. **Import-time settings guard**: Wrapped `settings` import in try/except
   at file top level to handle missing `.env` gracefully with a clean error
   message instead of an unhandled traceback. This deviates from the standard
   import pattern but is necessary because `settings` loads at import time.

2. **`send_info()` not `send_alert()` for success**: Success notification uses
   `send_info()` (Telegram only) because Gmail OAuth may not be set up yet.
   Failures use `send_alert()` (both channels) because failures matter more
   and the fallback logging in `send_alert()` handles missing credentials.

3. **ATR uses `ewm(span=period)`**: Standard Wilder smoothing via pandas EWM.
   This is a temporary implementation; Phase 2 replaces it with pandas-ta.

4. **Idempotency via position check**: Instead of crashing on re-run when a
   position already exists, the script checks first and skips the trade.
   This makes the milestone demo-friendly without hiding real errors.

5. **`os.makedirs("data", exist_ok=True)`**: Ensures the `data/` directory
   exists before any database operations. Without this, first-run on a fresh
   clone would fail at `validate_data()`.

6. **`compute_atr()` as module-level function**: Defined at module level
   (not inside `main()`) so it has a proper docstring, type hints, and can
   be tested independently if needed. It will not be imported by other modules.

7. **Stop-loss 3% hard cap applied**: The strategy rules specify a 3% hard
   cap on stop-loss distance. This is enforced even in the dry-run to match
   the production behaviour that will be implemented in Phase 2.
