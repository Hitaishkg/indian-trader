## 1. Purpose

`src/data/fetcher.py` is the OHLCV data acquisition layer for the Indian Trader pipeline. It fetches historical and recent OHLCV price data for NSE-listed stocks and sector indices, caches results as CSV files to avoid redundant network calls, and returns normalised DataFrames that conform exactly to the contract defined in `src/data/validator.py` Section 5.1. The output of this module can be passed directly to `validate_data()` without any additional transformation.

This module is consumed by:
- `src/data/cleaner.py` (Phase 1, step 4) -- receives raw fetched data for further cleaning
- The Data Collector Agent (Phase 4) -- calls `fetch_ohlcv` at 22:00 IST nightly
- `src/backtest/runner.py` (Phase 2) -- fetches historical data for backtesting

This module has zero knowledge of strategy logic, indicators, or fundamentals. It only fetches and normalises OHLCV price data.

## 4. DataFrame Output Contract

The output DataFrame must exactly match the validator.py Section 5.1 contract. The Coder Agent must ensure that the output of `fetch_ohlcv()` can be passed directly to `validate_data()` as the `ohlcv_df` argument without any additional transformation.

### Required columns (exact names, case-sensitive)

| Column | dtype | Description |
|---|---|---|
| `symbol` | `str` (object) | NSE ticker symbol, uppercase, without `.NS` suffix. E.g. `"RELIANCE"`, not `"RELIANCE.NS"` |
| `date` | `datetime64[ns, Asia/Kolkata]` | Trading date, timezone-aware IST. Time component is midnight IST |
| `open` | `float64` | Open price in INR |
| `high` | `float64` | High price in INR |
| `low` | `float64` | Low price in INR |
| `close` | `float64` | Close price in INR |
| `volume` | `float64` | Traded volume. Must be float64, not int64 |

The DataFrame must have a default integer index (RangeIndex), not a DatetimeIndex.

When fetching multiple symbols, all symbols are stacked into a single DataFrame with the `symbol` column distinguishing them.

## 5. Cache Design

### File naming convention

One CSV file per symbol per source:

```
data/cache/{SYMBOL}_{SOURCE}.csv
```

Examples:
- `data/cache/RELIANCE_yfinance.csv`
- `data/cache/RELIANCE_jugaad.csv`
- `data/cache/^CNXIT_yfinance.csv` (sector indices -- yfinance only)

The symbol in the filename is the raw NSE symbol (for stocks) or the yfinance ticker (for indices). The source is either `yfinance` or `jugaad`.

### Cache expiry

Cache freshness is determined by file modification time (`os.path.getmtime`):

```
cache_is_fresh = (time.time() - os.path.getmtime(cache_path)) < (cache_expiry_hours * 3600)
```

Default expiry: 24 hours for daily OHLCV data. Configurable via the `cache_expiry_hours` parameter on all public functions.

### Cache hit behaviour

If a cache file exists and is fresh:
1. Read the CSV into a DataFrame
2. Parse the `date` column as timezone-aware datetime (Asia/Kolkata)
3. Cast `volume` to float64
4. Return without making any network call
5. Log at INFO level: `"Cache hit for {symbol} from {source}"`

### Cache miss behaviour

If no cache file exists or the cache is expired:
1. Fetch from the primary source (yfinance)
2. Normalise the raw output to the contract format
3. Write the normalised DataFrame to CSV at the cache path
4. Log at INFO level: `"Cache miss for {symbol}, fetched from {source}"`
5. Return the normalised DataFrame

### CSV format

When writing cache files, use `df.to_csv(path, index=False)`. The `date` column is written as ISO 8601 strings with timezone info. When reading cache files, parse dates with `pd.to_datetime(df["date"], utc=True).dt.tz_convert("Asia/Kolkata")`.

## 6. Public API

### 6.1 `fetch_ohlcv`

```python
def fetch_ohlcv(
    symbols: list[str],
    start_date: datetime.date,
    end_date: datetime.date,
    cache_expiry_hours: int = 24,
) -> pd.DataFrame:
    """Fetch OHLCV data for one or more NSE stock symbols.

    Tries yfinance first, falls back to jugaad-data on failure.
    Results are cached as CSV files in data/cache/ with configurable expiry.
    Returns a single normalised DataFrame with all symbols stacked vertically.

    Args:
        symbols: List of NSE ticker symbols without .NS suffix.
                 E.g. ["RELIANCE", "TCS", "INFY"].
        start_date: Start date (inclusive) for historical data.
        end_date: End date (inclusive) for historical data.
        cache_expiry_hours: Cache freshness threshold in hours. Default 24.
                           Set to 0 to force a fresh fetch (bypass cache).

    Returns:
        pd.DataFrame conforming to the validator.py Section 5.1 contract:
        columns [symbol, date, open, high, low, close, volume],
        date is datetime64[ns, Asia/Kolkata], volume is float64.
        Sorted by (symbol, date) ascending.

    Raises:
        FetchError: If both yfinance and jugaad-data fail for any symbol.
                    The error message identifies the symbol and both failure reasons.
        ValueError: If symbols list is empty.
    """
```

Symbols are processed sequentially, not in parallel. Each symbol is fetched independently: if RELIANCE succeeds but TCS fails both sources, `FetchError` is raised for TCS. No partial results are returned -- the function succeeds completely or raises.

## 7. Source Fallback Logic

The fallback logic applies to `fetch_ohlcv` only. Sector indices (`fetch_sector_indices`) use yfinance exclusively because jugaad-data's `index_df` API is unreliable.

### For each symbol in `fetch_ohlcv`:

```
1. Check cache for {symbol}_yfinance.csv
   - If cache is fresh: return cached data. Done.

2. Try yfinance:
   a. Call yf.Ticker("{symbol}.NS").history(start=..., end=...)
   b. Validate result: not empty, close column has no all-NaN values
   c. If valid: normalise, write to {symbol}_yfinance.csv cache, return. Done.
   d. If invalid or exception: log WARNING with error detail, proceed to step 3.

3. Check cache for {symbol}_jugaad.csv
   - If cache is fresh: return cached data. Done.

4. Try jugaad-data:
   a. Call stock_df(symbol, start_date, end_date, series="EQ")
   b. Validate result: not empty, CLOSE column has no all-NaN values
   c. If valid: normalise, write to {symbol}_jugaad.csv cache, return. Done.
   d. If invalid or exception: log ERROR with error detail, proceed to step 5.

5. Raise FetchError with message:
   "Failed to fetch OHLCV for {symbol}. "
   "yfinance error: {yf_error}. "
   "jugaad-data error: {jd_error}."
```

The Coder Agent must store the yfinance exception/reason and the jugaad-data exception/reason separately so that the `FetchError` message includes both.

## 8. yfinance Normalisation

yfinance's `Ticker.history()` returns a DataFrame with:
- DatetimeIndex named `"Date"` with dtype `datetime64[s, Asia/Kolkata]` (already timezone-aware)
- Columns: `Open`, `High`, `Low`, `Close`, `Volume`, `Dividends`, `Stock Splits`

Normalisation steps (in order):

```python
# 1. Reset the DatetimeIndex to get a 'Date' column
df = raw_df.reset_index()

# 2. Select and rename columns to lowercase
df = df.rename(columns={
    "Date": "date",
    "Open": "open",
    "High": "high",
    "Low": "low",
    "Close": "close",
    "Volume": "volume",
})
df = df[["date", "open", "high", "low", "close", "volume"]]

# 3. Add the symbol column (NSE symbol WITHOUT .NS suffix)
df["symbol"] = symbol  # e.g. "RELIANCE", not "RELIANCE.NS"

# 4. Ensure date is timezone-aware Asia/Kolkata
# yfinance Ticker.history() already returns Asia/Kolkata-aware dates.
# If for any reason the dates are timezone-naive, localise them:
if df["date"].dt.tz is None:
    df["date"] = df["date"].dt.tz_localize("Asia/Kolkata")

# 5. Cast volume to float64 (yfinance returns int64 from Ticker.history)
df["volume"] = df["volume"].astype("float64")

# 6. Reorder columns to match contract
df = df[["symbol", "date", "open", "high", "low", "close", "volume"]]
```

Important: use `yf.Ticker(f"{symbol}.NS").history(start=..., end=...)`, NOT `yf.download()`. The `Ticker.history()` method returns single-level columns and timezone-aware dates. The `yf.download()` method returns multi-level columns and timezone-naive dates, requiring more complex handling.

## 9. jugaad-data Normalisation

jugaad-data's `stock_df()` returns a DataFrame with columns:

```
DATE, SERIES, OPEN, HIGH, LOW, PREV. CLOSE, LTP, CLOSE, VWAP,
VOLUME, VALUE, NO OF TRADES, DELIVERY QTY, DELIVERY %, SYMBOL
```

The `DATE` column is `datetime64[ns]` (timezone-naive) with time component at 18:30:00 (UTC representation of midnight IST).

Normalisation steps (in order):

```python
from jugaad_data.nse import stock_df

# 1. Fetch raw data
raw_df = stock_df(symbol, start_date, end_date, series="EQ")

# 2. Select and rename columns
df = raw_df.rename(columns={
    "DATE": "date",
    "OPEN": "open",
    "HIGH": "high",
    "LOW": "low",
    "CLOSE": "close",
    "VOLUME": "volume",
    "SYMBOL": "symbol",
})
df = df[["symbol", "date", "open", "high", "low", "close", "volume"]]

# 3. Normalise the date column:
#    jugaad-data returns dates with 18:30 time component (UTC offset artefact).
#    Normalise to midnight, then localise to Asia/Kolkata.
df["date"] = pd.to_datetime(df["date"]).dt.normalize()
df["date"] = df["date"].dt.tz_localize("Asia/Kolkata")

# 4. Cast volume to float64
df["volume"] = df["volume"].astype("float64")

# 5. Sort by date ascending
df = df.sort_values("date").reset_index(drop=True)
```

Note: jugaad-data returns rows in reverse chronological order (newest first). The sort in step 5 corrects this.

## 10. Error Handling

### Custom exception

```python
class FetchError(Exception):
    """Raised when OHLCV data cannot be fetched from any source.

    Attributes:
        symbol: The NSE ticker symbol that failed.
        yfinance_error: Description of the yfinance failure, or None if yfinance was not attempted.
        jugaad_error: Description of the jugaad-data failure, or None if jugaad-data was not attempted.
    """

    def __init__(
        self,
        symbol: str,
        yfinance_error: str | None = None,
        jugaad_error: str | None = None,
    ) -> None:
        self.symbol = symbol
        self.yfinance_error = yfinance_error
        self.jugaad_error = jugaad_error
        parts = [f"Failed to fetch OHLCV for {symbol}."]
        if yfinance_error:
            parts.append(f"yfinance error: {yfinance_error}.")
        if jugaad_error:
            parts.append(f"jugaad-data error: {jugaad_error}.")
        super().__init__(" ".join(parts))
```

### Exception handling rules

| Situation | Exceptions to catch | Action |
|---|---|---|
| yfinance network/API failure | `requests.exceptions.RequestException`, `ConnectionError`, `TimeoutError`, `ValueError`, `KeyError` | Log WARNING, proceed to jugaad-data fallback |
| yfinance returns empty DataFrame | (no exception -- check `df.empty`) | Log WARNING, proceed to jugaad-data fallback |
| yfinance close column is all NaN | (no exception -- check `df["Close"].isna().all()`) | Log WARNING, proceed to jugaad-data fallback |
| jugaad-data network/API failure | `requests.exceptions.RequestException`, `ConnectionError`, `TimeoutError`, `ValueError`, `KeyError` | Log ERROR, raise FetchError |
| jugaad-data returns empty DataFrame | (no exception -- check `df.empty`) | Log ERROR, raise FetchError |
| Cache CSV is corrupt/unreadable | `pd.errors.ParserError`, `pd.errors.EmptyDataError`, `ValueError` | Log WARNING, delete the corrupt file, proceed as cache miss |
| Empty symbols list | (no exception) | Raise `ValueError("symbols list must not be empty")` immediately |

No bare `except` clauses. Every `except` block catches a specific exception type or a tuple of specific types.

## 15. Explicit Non-Goals

This module does NOT:

- Fetch fundamentals data (ROE, D/E, EPS). That is `src/data/fundamentals.py` (Phase 1, step 5).
- Calculate technical indicators (RSI, MACD, ATR). That is `src/indicators/technical.py` (Phase 2).
- Apply any strategy logic (quality filter, momentum rank, regime filter).
- Write to SQLite or any database. All caching is CSV-only in `data/cache/`.
- Send notifications via Telegram or Gmail.
- Validate data quality. It produces normalised DataFrames; `src/data/validator.py` validates them.
- Clean or repair data (handle missing values, fill gaps). That is `src/data/cleaner.py` (Phase 1, step 4).
- Handle authentication tokens or broker API credentials.
- Perform parallel or async fetching. All fetches are sequential.
- Use `yf.download()`. It exclusively uses `yf.Ticker().history()` for cleaner output format.
- Fetch intraday or minute-level data. Only daily OHLCV.

def _fetch_jugaad(

## 17. yfinance `end_date` Handling

yfinance's `Ticker.history(start=..., end=...)` treats `end` as exclusive. To include `end_date` in the results, the Coder Agent must pass `end + timedelta(days=1)` to yfinance:

```python
import datetime
yf_end = end_date + datetime.timedelta(days=1)
df = yf.Ticker(f"{symbol}.NS").history(start=start_date, end=yf_end)
```

jugaad-data's `stock_df(symbol, from_date, to_date)` treats both dates as inclusive. No adjustment needed.

## 18. Sector Index Normalisation

Sector indices are fetched via `yf.Ticker(yf_ticker).history(...)` using the tickers from `SECTOR_INDEX_MAP`. The normalisation is identical to stock normalisation (Section 8) except:

- The `symbol` column is set to the human-readable key from `SECTOR_INDEX_MAP` (e.g. `"NIFTY_IT"`), not the yfinance ticker (e.g. `"^CNXIT"`).
- There is no jugaad-data fallback for indices. If yfinance fails, `FetchError` is raised immediately.
- Cache files use the yfinance ticker in the filename (e.g. `^CNXIT_yfinance.csv`).
